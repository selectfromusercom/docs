# 사용자 초대 및 관리

셀렉트 어드민에서 팀원을 초대하고, 역할에 맞는 권한을 부여하는 방법을 안내합니다.

## 시작하기 전에

멤버 초대는 **관리자(어드민 편집 + 계정 관리)** 권한을 가진 사용자만 가능합니다.  
초대받는 사람이 셀렉트에 가입하지 않은 이메일이어도 초대할 수 있으며, 최초 로그인 시 가입 절차가 진행됩니다.

## 초대 방법

### 1. 계정 페이지 이동

**관리자 → 사용자** 메뉴로 이동합니다.

> **편집** 또는 **관리자** 권한이 있어야 사용자 페이지에 접근할 수 있습니다.

### 2. 초대 버튼 클릭

사용자 관리 영역의 **초대** 버튼을 클릭합니다.

### 3. 초대 정보 입력

**새로운 사용자 초대** 모달에서 아래 항목을 입력합니다.

| 항목 | 설명 |
|------|------|
| **이메일 주소** | 초대할 팀원의 이메일. 소셜 계정 또는 이메일 OTP로 로그인 가능 |
| **알림 메일 발송** | 체크 시 초대 안내 이메일이 발송됨 (기본 체크) |
| **권한** | 부여할 역할 선택 (아래 권한 설명 참고) |

입력 완료 후 **초대** 버튼을 클릭합니다.

## 권한 종류

| 권한 | 설명 |
|------|------|
| **조회** (어드민 사용자) | 어드민 접속 및 사용 가능. 편집/데이터/로그/설정 접근 불가 |
| **수정** (어드민 편집) | 어드민 접속 및 사용/편집. 허용된 데이터/로그 가능. 설정 접근 불가 |
| **관리자** | 모든 기능과 설정 가능. 팀원 초대 포함 |
| **결제 담당자** | 카드 등록, 플랜 관리 가능. 어드민 접속 제외 |
| **권한 없음** | 계정은 유지하되 접근 권한 없음 (비활성화 용도) |

::: tip 권한 설정 팁
- 개발자나 데이터 분석 담당자에게는 **수정** 권한을 권장합니다.
- 프로젝트 관리자에게는 **관리자** 권한을 부여하세요.
- 최종 사용자 팀원은 **조회** 권한이 적합합니다. (어드민 화면 접근, 사용 가능)
:::

## 환경별 접근 제한 (정책 설정)

초대 후 사용자 목록에서 각 멤버에게 **환경 접근 정책**을 추가할 수 있습니다.

- **정책 없음**: Production, Development 환경 모두 접근 가능
- **Production 체크**: 운영 환경만 접근 허용
- **Development 체크**: 개발/테스트 환경만 접근 허용

정책 열에서 **클릭하여 환경별 정책 추가**를 선택해 설정합니다.

## 리소스 및 쿼리 단위 권한

사용자별로 **리소스**나 **쿼리**에 접근 세부 권한을 지정할 수 있습니다.

- 사용자 관리 아래 탭에서 **리소스** 탭을 선택합니다.
- 각 리소스/쿼리에 대해 체크박스로 접근 허용 여부를 설정합니다.

## 사용자 관리

### 권한 변경

사용자 목록에서 권한 드롭다운을 클릭해 즉시 변경할 수 있습니다.

### 비활성화

접근을 차단하되 계정을 유지하려면 권한을 **권한 없음**으로 변경합니다.

### 사용자 삭제

권한 드롭다운 최하단의 **사용자 권한 삭제**를 클릭하면 해당 사용자를 팀에서 제거합니다.

## 자주 묻는 질문

**Q. 셀렉트에 가입하지 않은 이메일도 초대할 수 있나요?**  
네. 초대 이메일을 발송하면 초대받은 사람이 링크를 통해 가입 절차를 진행합니다.

**Q. 구글 계정이 아니어도 로그인할 수 있나요?**  
이메일 OTP 로그인을 지원합니다. 소셜 계정과 이메일 OTP 중 선택할 수 있습니다.

**Q. 무료 플랜에서 초대할 수 있는 인원은?**  
무료 플랜 기준 최대 **5명**(본인 포함)까지 사용자를 추가할 수 있습니다.

**Q. 초대 메일을 받지 못했어요.**
스팸함을 확인하거나, 관리자가 사용자 관리 페이지에서 재초대를 진행해 주세요.

**Q. 초대 링크로만 접속 가능한가요?**
아니요. 셀렉트 어드민 주소에서 로그인하거나, 지정된 개별 도메인 링크로 접속할 수 있어요.
